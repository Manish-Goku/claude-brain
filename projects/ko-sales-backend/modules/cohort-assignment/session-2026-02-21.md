# Session: Cohort System — 2026-02-21 (updated 2026-03-05)

## What Was Built

### Core Cohort CRUD
- `POST /cohorts` — create with rule validation + auto-evaluation
- `GET /cohorts` — paginated list with type/search filters
- `GET /cohorts/fields` — 40+ fields from FIELD_REGISTRY for frontend dropdown
- `GET /cohorts/:id` — single cohort
- `GET /cohorts/:id/members` — paginated pii_ids
- `GET /cohorts/:id/progress` — static cohort drift tracking
- `POST /cohorts/:id/recalculate` — re-evaluate members
- `PUT /cohorts/:id` — update (auto re-evaluates if rules change)
- `DELETE /cohorts/:id` — delete

### Rule Engine (Dual Path)
- **Field Registry:** 40+ fields across lead/contact/customer with auto entity_scope detection
- **Rule Compiler:** json-rules-engine format → MongoDB queries (12 operators)
- **Rule Evaluator:**
  - Fast: MongoDB aggregation with $lookup joins by pii_id
  - Slow: json-rules-engine batch evaluation (500/batch) for unsupported operators
  - Single: in-memory match or engine run for dynamic cohort updates

### Dynamic Cohort Updates
- `CohortEvaluatorService` listens to `entity.updated` events (via EventEmitter2)
- Finds overlapping dynamic cohorts by `rule_fields` intersection
- Adds/removes entity from cohort members

### BullMQ Assignment Infrastructure
- `@nestjs/bullmq` + `bullmq` with Redis
- `BullModule.forRootAsync()` in `app.module.ts`
- Queue: `cohort-assignment`, Job: `assign-cohort-leads`

### Cohort Assignment Feature
- `POST /cohorts/:id/assign` — sync (<100 leads) or async (>=100)
- `GET /cohorts/assign-jobs/:job_id` — job status with progress
- **Strategies:** `round_robin` (default), `capacity_weighted`
- Job tracking in `cohort_assignment_jobs` collection (CAJ-N)
- Progress updates every 10 leads

### Capacity-Weighted Distribution
- `Agent.daily_capacity` field (default 50)
- `agents.service.get_agent_capacity(agent_id)`
- Proportional: `share = (capacity / total_capacity) * total_leads`
- Remainder-based rounding for exact totals

## Bugs Found & Fixed (2026-03-05 testing session)

### 1. Operator validation missing in rule-compiler
- `validate_node()` checked fact + value but NOT operator name
- Invalid operator (e.g. `"foobar"`) passed validation → caused server hang during evaluation
- **Fix:** Added `OPERATOR_MAP` + `UNSUPPORTED_OPERATORS` check in `validate_node()`

### 2. BSON 16MB overflow on large cohorts
- `evaluate_cohort()` stored ALL matching pii_ids in `members` array
- With 789k active leads, the document exceeded MongoDB 16MB BSON limit
- **Fix:** Capped `members` array at 500k, `member_count` still reflects true total

### 3. ImapFlow crash killing server (unrelated, not fixed)
- `EmailPollService` cron triggers ImapFlow socket timeout → unhandled error event
- Crashes entire Node.js process
- **Workaround:** Temporarily disabled `CustomerSuccessModule` during testing
